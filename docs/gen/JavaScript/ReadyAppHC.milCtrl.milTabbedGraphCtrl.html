Class: milTabbedGraphCtrl

ReadyAppHC.milCtrl. milTabbedGraphCtrl

new milTabbedGraphCtrl()

Acts as the interface for all controllers that need tabs for day, week, month, and year views. This interface defines variables that identify tabs and associates those tabs with a particular kind of graph directive. (For example, the day route is to the integer 100, and 100 is to bar graphs.) This interface assists with routing for each tab, so you should set the baseRoute property in your controller's initializer so that this interface knows how to build routes when a tab is pressed. A page reload will not be triggered when this interface creates a new route for tabs; in fact, reloads are manually surpressed by this class. The route change exists solely for the native code to inject new JSON data when a new tab is pressed.
Properties:
Name Type Description
baseRoute string Defines the basis for routes that are generated by this interface when a new tab is pressed. For example, if the route /WebView/foo is provided, then the routes /WebView/foo_day, /WebView/foo_week, /WebView/foo_month, and /WebView/foo_year are generated by this interface when the appropriate tab is pressed.
displayType integer Defines the type of graphs that should be displayed, where the integer 0 represents bar graphs, and the integer 1 represnts line graphs. This should be used in conjunction with an ng-switch statement in the HTML code. (For example, when 0, define a bunch of bar-graph directives.)
selectedTab integer Defines the tab that is selected, where the integer 48 represents days, the integer 14 represents weeks, the integer 8 represents months, and the integer 24 reprsents years.
dayVal integer Defined by the integer 48.
weekVal integer Defined by the integer 14.
monthVal integer Defined by the integer 8.
yearVal integer Defined by the integer 24.
