Class: milSyncedGraphCtrl

ReadyAppHC.milCtrl.milTabbedGraphCtrl. milSyncedGraphCtrl

new milSyncedGraphCtrl()

Acts as the interface for all controllers that use graph directives. In Physio, this includes bar-graph.js, line-graph.js, and time-graph.js. milSyncedGraphCtrl provides functionality that allows graphs to scroll and work synchronously with other graphs that exist in the same controller. Using a graph directive in a controller that isn't a decendent of this class will cause the graph to exhibit undefined behavior.
Properties:
Name Type Description
transformData object An object with only one key, xGraph, that's used to determine the x-transform to apply to each graph. This property has a two-way binding with bar-graph and line-graph directives. When a graph requests a transform (that is, it had a pan gesture put upon it), it modifies the xGraph value in this object. All other graphs are listening for changes on this property (hence the two-way binding) and will update themselves accordingly.
iter number An iterator that allows graphs to consume data in the order that they appear. (See the nom function to learn more.)
data array An array of JSON data to be consumed by each graph. Graphs will consume data in this array from the top down. This means that the first piece of data in this array will be consumed by the first graph from the top, and the last piece of data in this array will be consumed by the first graph from the bottom.

(static) nom()

Called by a graph when it wants to consume data. Implementations of bar-graph and line-graph call this function at the point where they want to consume JSON data (ie, when the graph is told to invalidate via a broadcast). This function uses the iter property to determine what element in the data array needs to be given to the graph.
